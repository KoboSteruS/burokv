# Политика безопасности

## Важные настройки для production

### 1. SECRET_KEY

**КРИТИЧЕСКИ ВАЖНО**: Никогда не используйте дефолтный SECRET_KEY в production!

Генерация нового ключа:
```bash
python -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"
```

### 2. DEBUG

Всегда устанавливайте `DEBUG=False` в production:
```env
DEBUG=False
```

### 3. ALLOWED_HOSTS

Укажите только ваши домены:
```env
ALLOWED_HOSTS=yourdomain.com,www.yourdomain.com
```

### 4. База данных

- Используйте PostgreSQL вместо SQLite в production
- Используйте сильные пароли
- Ограничьте доступ к БД только с localhost

### 5. SSL/HTTPS

- Обязательно используйте HTTPS
- Настройте автоматическое обновление сертификатов Let's Encrypt

### 6. Файлы .env

- Никогда не коммитьте .env файлы в git
- Храните секреты в безопасном месте
- Используйте разные ключи для разных окружений

### 7. Права доступа

```bash
# Правильные права для файлов
find . -type f -exec chmod 644 {} \;
find . -type d -exec chmod 755 {} \;

# Права для скриптов
chmod 755 manage.py deploy.sh

# Права для медиа
chmod -R 755 media/
```

### 8. Резервное копирование

Настройте регулярное резервное копирование:
- База данных (ежедневно)
- Медиа файлы (еженедельно)
- Конфигурационные файлы

### 9. Мониторинг

- Настройте логирование ошибок
- Мониторьте использование ресурсов
- Настройте алерты при проблемах

### 10. Обновления

- Регулярно обновляйте зависимости
- Следите за уязвимостями в пакетах
- Используйте `pip list --outdated` для проверки
